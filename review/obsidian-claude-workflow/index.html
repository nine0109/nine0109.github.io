Obsidian + Claude Code 워크플로우

Obsidian에서 노트를 작성하고, Claude Code가 편집/보강하는 협업 워크플로우입니다.

레포지토리 구조 (Submodule)

레포지토리용도경로
nine0109/notes노트 전용 (Obsidian, 모바일)독립 관리
nine0109/nine0109.github.io사이트 전체_notes/ → submodule

Submodule 적용 브랜치

브랜치상태
main적용됨
develop적용됨

장점

  • 아이폰 Working Copy에서 notes 레포만 클론하면 됨
  • 노트와 사이트 코드 분리
  • 각각 독립적으로 커밋/푸시 가능
  • main/develop 모두 동일한 submodule 구조

환경별 설정

데스크탑 (Obsidian + Claude Code)

항목설정
Vault 경로~/Documents/GitHub/notes
플러그인Obsidian Git
자동 백업활성화 권장

모바일 (Working Copy)

항목설정
클론 레포nine0109/notes
브랜치main

요청 방식

문서 내에 Obsidian 주석으로 Claude에게 요청:

%%cl: 요청 내용%%

요청 예시

%%cl: 이 문서 오타 수정해줘%%
%%cl: 1, 2, 3 섹션 내용 보강해줘%%
%%cl: 영어로 번역해줘%%
%%cl: 태그 정리해줘%%

장점

  • Obsidian에서 %%는 주석 처리되어 미리보기에 표시 안 됨
  • 짧고 입력하기 쉬움

워크플로우 흐름

1. [Obsidian/모바일] 노트 작성/수정
2. [Obsidian/모바일] %%cl: 요청%% 추가
3. [Git] 커밋 & 푸시 (notes 레포)
4. [Claude Code] "XX 문서 수정했어" 알림
5. [Claude Code] 요청 확인 및 처리
6. [Claude Code] 문서 수정 완료 & 푸시
7. [Obsidian/모바일] pull로 변경 내용 확인

사용 가능한 요청 유형

유형설명예시
문서 정리구조, 포맷팅 개선섹션 정리, 표 추가
내용 보강설명 추가, 예시 보완상세 설명 추가
오타 수정맞춤법, 문법 교정오탈자 수정
번역한/영 번역영문 번역
태그 정리태그 추가/수정관련 태그 추천
코드 검토코드 블록 검토코드 개선 제안

퍼블리싱 흐름

1. notes 레포 변경 푸시

cd ~/Documents/GitHub/notes
git add -A && git commit -m "update notes" && git push

2. 메인 사이트에서 submodule 업데이트

cd ~/Documents/GitHub/nine0109.github.io
git submodule update --remote _notes
git add _notes
git commit -m "update notes submodule"
git push

주의사항

  • notes 레포와 메인 레포는 별도로 커밋/푸시 필요
  • 사이트 배포 시 submodule 업데이트 필수
  • 요청 주석(%%cl: %%)은 처리 후 자동 삭제됨

리뷰 완료 후

문서 확인이 완료되면 Claude에게 다음과 같이 요청하세요:

"리뷰 완료, 발행해줘"
