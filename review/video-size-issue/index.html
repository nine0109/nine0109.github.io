GitHub Pages 비디오 재생 문제 해결

비디오 재생 문제 해결 방안

문제상황

GitHub Pages에서 큰 비디오 파일이 재생되지 않음

파일 크기 현황

  • ✅ Eyisha_Zyer_1_Combine_photos_into_new_scenes.mp4 - 0.7MB (정상)
  • ✅ Eyisha_Zyer_2_Can_edit_image_by_describing_to_it.mp4 - 1MB (정상)
  • ✅ Eyisha_Zyer_5_Move_styles_from_one_image_to_another.mp4 - 0.8MB (정상)
  • ✅ Eyisha_Zyer_9_A_new_era_of_indie_anime_animators.mp4 - 2.4MB (정상)
  • ❌ Eyisha_Zyer_4_Start_frames_for_Ads.mp4 - 3MB (경계선)
  • ❌ Eyisha_Zyer_3_Nano_Banana_Runway_Act_2.mp4 - 4.3MB (큰 파일)
  • ❌ Eyisha_Zyer_17_Cartoon_Animation.mp4 - 9.2MB (매우 큰 파일)

해결책

1. 즉시 적용 (완료)

  • muted 속성 추가하여 자동재생 정책 문제 해결

2. 큰 파일 처리 방안

A. 파일 압축 (권장)

# ffmpeg 설치 필요
brew install ffmpeg

# 압축 명령어
ffmpeg -i input.mp4 -vf scale=720:-1 -c:v libx264 -crf 28 -c:a aac -b:a 128k output.mp4

B. 대체 링크 사용

큰 파일들을 X 링크로 대체:

<!-- 기존 -->
![](/assets/img/2025-08-31_NanoBanana/Eyisha_Zyer_17_Cartoon_Animation.mp4)

<!-- 대체 -->
[🎥 17. Cartoon Animation 영상 보기](https://x.com/i/status/원본링크)

C. GIF 변환

비디오를 GIF로 변환하여 용량 줄이기

권장사항

  1. 3MB 이하 파일들은 그대로 유지
  2. 3MB 이상 파일들은 압축 또는 링크로 대체
  3. 특히 9.2MB 파일은 반드시 처리 필요

테스트 필요

  • 로컬에서 정상 재생되는지 확인
  • GitHub Pages에서 재생 확인
  • 모바일에서 재생 확인
