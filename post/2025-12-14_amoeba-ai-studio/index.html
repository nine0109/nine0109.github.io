ai 서비스

amoeba AI Studio를 만들었습니다

사내에서 사용할 AI 통합 작업 환경을 설계하고 구현했습니다. 이름은 amoeba AI Studio입니다.

현재는 외부에 공개된 서비스는 아니며, 사내 네트워크에서만 접근 가능한 내부 전용 AI 플랫폼으로 운영 중입니다. 이 글에서는 실제 접근 주소 대신, 어떤 문제를 해결하기 위해 만들었는지와 주요 기능을 정리합니다.

제작 배경

업무에서 AI를 사용하는 빈도가 높아질수록 다음과 같은 문제가 반복되었습니다.

  • 이미지, 비디오, 텍스트 AI가 각각 다른 서비스에 분산되어 있음
  • 모델마다 인터페이스와 프롬프트 방식이 달라 작업 흐름이 자주 끊김
  • 개인 단위로 사용된 프롬프트와 결과물이 조직 차원에서 축적되지 않음

이 문제를 해결하기 위해 여러 AI 모델을 한 환경에서 통합해 사용할 수 있는 사내 전용 플랫폼을 직접 만들었습니다.

목표

User Goal

한곳에서 끝내는 생산성 향상

이미지 생성, 비디오 제작, 3D, 전문 AI 채팅까지 여러 도구를 오가며 작업하지 않고, 하나의 인터페이스에서 바로 실행할 수 있도록 구성했습니다.

툴 전환 비용을 최소화하고, 사용자는 결과물과 의사결정에만 집중할 수 있도록 설계했습니다.

Business Goal

프롬프트 데이터의 자산화

사내에서 발생하는 AI 프롬프트와 생성 결과를 일회성 사용으로 끝내지 않고 데이터 자산으로 축적하는 것을 목표로 했습니다.

최신 AI 모델을 단순히 나열하는 것이 아니라, 업무 목적에 맞게 배치하고 사용량을 관리할 수 있도록 설계했습니다.

주요 기능

1. 목적별 AI 에이전트 (Multi-Agent Chat)

업무 성격에 따라 AI의 역할을 분리했습니다.

  • Agent 사규 및 내부 문서를 기준으로 답변하는 기업 맞춤형 AI

  • Project AI 업로드된 프로젝트 문서와 채팅 기록을 종합해 판단하는 AI

  • Design Chat 디자인 트렌드와 UI/UX 설계에 특화된 AI 어시스턴트

단일 챗봇이 아닌, 업무 목적에 따라 선택하는 구조로 구성했습니다.

2. 멀티모달 생성 환경

텍스트, 이미지, 비디오, 3D 생성 기능을 하나의 환경에서 제공합니다.

  • 지원 모델
    • Google Gemini (2.5 / 3 / 3.1)
    • Imagen 4 (Fast / Standard / Ultra)
    • Flux 2 Pro
    • Veo 3 / Veo 3.1 (Preview / Fast / Standard)
    • WAN 2.5
    • Kling Edit

모델별 옵션과 Pre-prompt를 정리해 비율, 방향, 스타일 등을 비교적 정밀하게 제어할 수 있도록 구성했습니다.

생성에 그치지 않고 기존 결과물을 편집하거나 병합하는 Edit / Merge 모드도 지원합니다.

3. 협업 및 관리 기능

  • Project Sharing 프로젝트 단위로 AI 생성 과정과 결과물을 공유할 수 있도록 했습니다.

  • Trash System 삭제된 항목은 30일간 보관되며, 관리자 등급에 따라 실제 삭제 여부를 제어합니다.

  • Admin System 사용자별 미디어 타입별 생성 횟수 제한, AI 사용 현황을 확인할 수 있는 대시보드를 제공합니다.

4. 사용자 중심 인터페이스

  • 라이트 / 다크 테마 지원
  • 북마크, 좋아요 기반 필터링 및 레이팅
  • 그리드 뷰 아이템 수 조절, Fit / Fill 설정
  • 트렌드 / 인기순 / 최신순 정렬 지원

개인의 작업 방식에 맞게 화면을 조정할 수 있도록 구성했습니다.

마무리

amoeba AI Studio는 단기간에 만든 데모가 아니라, 사내에서 실제로 사용할 것을 전제로 설계한 내부 플랫폼입니다.

앞으로 모델 추가, 에이전트 고도화, 프롬프트 데이터 활용 방식 등을 계속 개선해 나갈 예정입니다.
